# ADR 0004: 双主题(夕阳/月夜) + 入场页 + localStorage 首访引导

- **状态**: Accepted
- **日期**: 2026-04-29
- **决策者**: LIYUAN

## Context

希望网站有视觉记忆点,而非千篇一律的浅色/深色主题。
经过设计讨论,确立了"双主题(夕阳 sunset / 月夜 moonlight)"的方向。

核心问题:**用户首次访问怎么决定用哪个主题?**

| 选项 | 优点 | 缺点 |
|---|---|---|
| 默认夕阳,角落放切换按钮 | 实现最简,符合"默认+设置"心智 | 缺乏仪式感,主题切换体验弱 |
| 强制选择(modal 拦截首访) | 强保证用户感知双主题 | 过度打扰,modal 体验差 |
| 跟随系统 `prefers-color-scheme` | 零打扰 | 系统是二值(light/dark),映射到夕阳/月夜不直观 |
| **入场页 `/welcome` 引导首选 + localStorage 记忆** | **首访仪式感强,回访零打扰** | 多一个路由 + 防闪烁要小心 |

第二个问题:**主题切换用什么技术实现?**(详见 ADR 0004)

第三个问题:**怎么避免主题切换时白屏闪烁?**

## Decision

- **入场页**: `src/pages/welcome.astro`
  - 全屏背景图 + 两个主题选择按钮(夕阳 / 月夜)
  - 用户选择后 → 写入 `localStorage.theme` → 跳转回 `/`
- **首访逻辑**: BaseLayout 头部内联脚本同步执行
  - 读 `localStorage.theme`
  - 若不存在且当前不在 `/welcome`,跳 `/welcome`
  - 若存在,在 DOM 渲染前设置 `<html data-theme="...">`
- **常驻切换按钮**: 全站导航栏右侧 ThemeToggle,任何时候可切
- **持久化**: `lib/theme.ts` 的 `applyTheme` 同时设 DOM 属性 + localStorage

## Consequences

**正面**:
- ✅ 首访有仪式感,双主题被用户明确感知
- ✅ 回访零打扰,直接进入上次选的主题
- ✅ 头部内联脚本同步执行,主题在首屏渲染前就确定,无白屏闪烁
- ✅ 全站任何时候可切换主题,无需回入场页

**负面 / 限制**:
- ❌ 多一个路由(welcome),站点结构略复杂
- ❌ 内联脚本与构建分离,要小心 SSR/SSG 阶段不能访问 localStorage
  (Astro 构建时不执行客户端脚本,实际无影响)
- ❌ 用户清缓存后会再次跳 welcome,可能产生"为什么又问我"的困惑(实际很少触发)

**未来变更成本**:
- 🔄 增加第三种主题(如"清晨")需扩展 CSS 变量 block + welcome 按钮 + ThemeToggle 选项
- 🔄 改"跟随系统"成本中等,需重写首访判断逻辑

## 验证依据

阶段 2.4 完成(2026-04-29),实测:
- 首访自动跳 `/welcome`、选完跳回首页、刷新仍是所选主题
- 切换按钮全站可用,持久化正确
- 无主题切换时的白屏闪烁
- localStorage 清空后首访逻辑重新触发(预期行为)
